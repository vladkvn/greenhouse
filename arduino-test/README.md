# Arduino test: датчик + LCD + ESP8266 (Wi‑Fi)

Два скетча: **Arduino Uno** снимает показание с **A0**, выводит на **LCD I²C (0x27)** и обменивается строками по UART с **модулем ESP8266** (Petoi). ESP подключается к домашней Wi‑Fi и даёт **HTTP API**: телеметрия и команды.

## Схема подключения

| ESP8266 | Arduino Uno |
|--------|-------------|
| **TX** | **D2** (приём `SoftwareSerial`, RX) |
| **RX** | **D3** (передача с Uno; **обязателен делитель 5 В → 3,3 В** на эту линию) |
| **GND** | **GND** |

Питание ESP — по документации модуля (3,3 В, достаточный ток). Общая земля с Uno обязательна.

## Протокол по UART (57600 бод, строки с `\n`)

| Направление | Формат | Назначение |
|-------------|--------|------------|
| Uno → ESP | `T,<число>` | Телеметрия (число — показание АЦП, обычно 0…1023) |
| ESP → Uno | `C,<ИМЯ>` | Команда из whitelist (см. ниже) |

Пустые и нераспознанные строки игнорируются.

### Разрешённые команды (одинаково в обоих скетчах)

- `PING` — отладка (в Serial Uno печатается `cmd:PING`).
- `RELAY_ON` / `RELAY_OFF` — виртуальное «реле» (флаг на Uno, на LCD строка 2: `R:ON` / `R:OFF`). Подключите реальное реле к цифровому пину и `digitalWrite` в обработчике команды при необходимости.

## Настройка Wi‑Fi на ESP8266

В `esp8266-web-bridge/esp8266-web-bridge.ino` задайте:

- `STASSID` — имя сети (SSID).
- `STAPSK` — пароль.

Либо при сборке передайте флаги компилятора `-DSTASSID=...` и `-DSTAPSK=...`.

**Не коммитьте** реальные пароли в открытый репозиторий.

### Опционально: токен для `/command`

Если задать непустой `COMMAND_TOKEN`, каждый запрос к `/command` должен содержать параметр `token` с этим значением (иначе `401`).

## Прошивка

1. **ESP8266** — в Arduino IDE: менеджер плат ESP8266, плата **Generic ESP8266 Module** (как в [документации Petoi](https://docs.petoi.com/communication-modules/wifi-esp8266)). Откройте каталог `esp8266-web-bridge/`, залейте `esp8266-web-bridge.ino` через USB‑адаптер Petoi.
2. **Uno** — откройте `arduino-test.ino`, выберите плату Arduino Uno, загрузите.

После первого подключения к Wi‑Fi IP адрес ESP печатается в **Serial Monitor** (57600) при старте (удобно, если USB к ESP ещё подключён).

## HTTP API (в одной LAN с ПК)

Подставьте IP ESP8266 (из Serial или админки роутера).

### Телеметрия

```http
GET http://<IP>/sensor
```

Ответ `200`:

```json
{"value":512,"ageMs":120}
```

`ageMs` — сколько миллисекунд прошло с последней принятой по UART строки `T,...`. Если Uno ещё не прислал данные — `503` и `{"error":"no_data_yet"}`.

Корень `GET /` перенаправляет на `/sensor`.

### Команда

```http
GET http://<IP>/command?cmd=RELAY_ON
```

или то же с `RELAY_OFF`, `PING`.

Если включён `COMMAND_TOKEN`:

```http
GET http://<IP>/command?cmd=PING&token=ВАШ_СЕКРЕТ
```

Ответ `200`:

```json
{"ok":true,"cmd":"RELAY_ON"}
```

Неверное или пустое имя команды — `400`. Без токена при включённом токене — `401`.

### Примеры с терминала (curl)

```bash
curl -s "http://192.168.1.50/sensor"
curl -s "http://192.168.1.50/command?cmd=RELAY_ON"
curl -s "http://192.168.1.50/command?cmd=RELAY_OFF"
curl -s -X POST -d "cmd=PING" "http://192.168.1.50/command"
```

Замените `192.168.1.50` на IP вашего ESP8266.

## Поведение на LCD (Uno)

- Строка 1: `S:<значение>` — показание с A0.
- Строка 2: `WET` или `DRY` (порог 500) и `R:ON` / `R:OFF` для виртуального реле.

## Расширение

- Добавьте новые имена в **whitelist** в обоих файлах и реализацию на Uno (пины, ШИМ и т.д.).
- Для надёжности в продакшене ограничьте доступ к HTTP (отдельная VLAN, firewall, сильный `COMMAND_TOKEN`).
